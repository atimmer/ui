# Project instructions

- Anton owns this project.
- Use `jj`, not `git`, for version-control operations.
- Use Vite+ commands (`vp`) for installs, formatting, linting, tests, builds, and package builds.
- Do not run the development server directly; use the local `devservers` CLI.
- Keep solutions concise and do not use TypeScript `any` unless it is unavoidable.
- Update `README.md` whenever setup, usage, scope, or release behavior changes.
- Add user-facing changes to the `Unreleased` section of `CHANGELOG.md`. Fold follow-up fixes into the original entry.
- Run `vp run ready` before publishing changes.
